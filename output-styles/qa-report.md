# Output Style: QA Report

Use for live QA, visual QA, test execution, and release checks.

## Shape

1. Overall: PASS, FAIL, or PASS with warnings.
2. Criteria table.
3. Bugs by severity.
4. Rework instructions.
5. Artifacts.

## Rules

- Do not soften a failing criterion.
- Every FAIL needs a reproduction path.
- Attach screenshot, log, or command evidence when available.
- Distinguish critical, major, minor, and info.
